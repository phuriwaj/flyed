# Decap CMS Integration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Decap CMS to the flyed Astro project so non-technical editors can author and publish bilingual (en + th) blog content through a browser GUI, while preserving the Git-based PR review workflow and Cloudflare Pages preview deploys.

**Architecture:** Decap CMS loads from a static `public/admin/index.html` shell. Decap Cloud handles OAuth and commits to PR branches (no backend). Content uses two-file i18n (`*.en.mdx` + `*.th.mdx`) wired through a new `blogTh` Astro collection. A one-shot migration script splits existing blog posts. Custom preview iframe loads CF Pages branch deploys. No new npm dependencies; Decap JS loaded from `unpkg.com`.

**Tech Stack:** Astro 7, Decap CMS 3.7+ (CDN), Vitest 2.x (unit), Playwright 1.61+ (E2E), Cloudflare Pages (preview deploys), gray-matter (migration script — needs install).

## Global Constraints

- Node ≥ 22.12.0 (per `package.json` engines)
- Decap CMS version: `^3.7.0` from `https://unpkg.com/decap-cms@^3.7.0`
- Cloudflare Pages project name: `flyed` (branch preview format: `https://<branch>--flyed-dev.pages.dev`)
- Repo: `flyed-dev/flyed`, main branch: `main`
- Decap path templates use `{{slug}}` (the default Decap entry slug), NOT `{{fields.slug}}`
- Decap config has `publish_mode: editorial_workflow` (PR-only, no direct main commits)
- Thai blog files start with `draft: true` after migration to hide untranslated content
- Frontmatter for both `blog` and `blogTh` keeps identical Zod schema (9 fields, no `*Th` suffix in blog)
- All admin pages (`/admin/*`) are public but noindex; auth happens in Decap Cloud before any Git write
- `astro check` and `npm run build` must pass after each task
- Conventional commit messages; commit frequently (after each task)

Create `src/lib/migrate-blog-i18n.ts`:

```ts
import { readdirSync, readFileSync, writeFileSync, renameSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import matter from 'gray-matter';

export interface MigrateResult {
  migrated: number;
  skipped: number;
  errors: string[];
}

export function migrateBlogI18n(dir: string): MigrateResult {
  const result: MigrateResult = { migrated: 0, skipped: 0, errors: [] };
  const files = readdirSync(dir);
  const mdxFiles = files.filter(
    (f) => f.endsWith('.mdx') && !f.endsWith('.en.mdx') && !f.endsWith('.th.mdx'),
  );

  for (const file of mdxFiles) {
    const srcPath = join(dir, file);
    const slug = file.replace(/\.mdx$/, '');
    const enPath = join(dir, `${slug}.en.mdx`);
    const thPath = join(dir, `${slug}.th.mdx`);

    try {
      const raw = readFileSync(srcPath, 'utf8');
      const parsed = matter(raw);

      // English version: original content, original draft flag
      const enContent = matter.stringify(parsed.content, parsed.data);
      const enTemp = `${enPath}.new`;
      writeFileSync(enTemp, enContent);
      renameSync(enTemp, enPath);

      // Thai version: copy frontmatter, prepend TODO, mark draft: true
      const thData = { ...parsed.data, draft: true };
      const thBody = `<!-- TODO(editor): translate this post to Thai. Set draft: false when done. -->\n\n${parsed.content}`;
      const thContent = matter.stringify(thBody, thData);
      const thTemp = `${thPath}.new`;
      writeFileSync(thTemp, thContent);
      renameSync(thTemp, thPath);

      // Delete original
      const srcTemp = `${srcPath}.new`;
      renameSync(srcPath, srcTemp);
      renameSync(srcTemp, srcPath.replace(/\.mdx$/, '.deleted.mdx'));
      // Actually delete the original
      const { unlinkSync } = require('node:fs') as typeof import('node:fs');
      unlinkSync(srcPath);

      result.migrated++;
    } catch (err) {
      result.errors.push(`${file}: ${err instanceof Error ? err.message : String(err)}`);
    }
  }

  // Count skipped (already-split files)
  result.skipped = files.filter(
    (f) => (f.endsWith('.en.mdx') || f.endsWith('.th.mdx')) && !mdxFiles.includes(f),
  ).length;

  return result;
}
```

**Interfaces:**

- [ ] **Step 1: Create `public/admin/preview.html`**

```html
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <meta name="robots" content="noindex,nofollow" />
    <title>flyed CMS — Preview</title>
    <style>
      * { box-sizing: border-box; }
      body { margin: 0; font-family: system-ui, -apple-system, sans-serif; height: 100vh; display: flex; flex-direction: column; }
      .preview-banner { padding: 8px 12px; background: #fef3c7; color: #78350f; font-size: 13px; border-bottom: 1px solid #fbbf24; flex-shrink: 0; }
      .preview-banner code { background: rgba(0,0,0,0.05); padding: 1px 4px; border-radius: 3px; }
      .preview-frame { width: 100%; flex-grow: 1; border: 0; background: #fff; }
      .preview-loading { display: flex; align-items: center; justify-content: center; height: 100%; color: #6b7280; font-size: 14px; }
    </style>
  </head>
  <body>
    <div class="preview-banner" id="banner">Loading preview…</div>
    <iframe id="frame" class="preview-frame" title="Post preview"></iframe>
    <script>
      (function () {
        var params = new URLSearchParams(window.location.search);
        var branch = params.get('branch') || 'main';
        var slug = params.get('slug') || '';
        var locale = params.get('locale') || 'en';

        // Strip any file extension Decap might pass in the slug
        var cleanSlug = slug.replace(/\.(en|th)\.mdx?$/, '').replace(/\.mdx?$/, '');

        var previewUrl;
        if (branch === 'main' || !branch) {
          previewUrl = 'https://flyed.dev/blog/' + cleanSlug;
        } else {
          // CF Pages branch deploy format: <branch>.<project>.pages.dev
          // Branch names from Decap may have slashes; replace with -- for subdomain
          var safeBranch = branch.replace(/\//g, '--');
          previewUrl = 'https://' + safeBranch + '--flyed-dev.pages.dev/blog/' + cleanSlug;
        }

        var frame = document.getElementById('frame');
        var banner = document.getElementById('banner');

        banner.innerHTML = branch === 'main'
          ? 'Live site: <code>main</code> branch'
          : 'Preview: <code>' + branch + '</code> — first load may take 30–60s while CF Pages deploys';

        frame.src = previewUrl;

        frame.addEventListener('load', function () {
          try {
            // Detect 404 in iframe
            var title = frame.contentDocument && frame.contentDocument.title;
            if (title && /404|not found/i.test(title)) {
              banner.innerHTML = 'Page not found at <code>' + previewUrl + '</code> — has the branch deployed yet?';
              banner.style.background = '#fee2e2';
              banner.style.color = '#7f1d1d';
            }
          } catch (e) {
            // Cross-origin; can't inspect. Assume success.
          }
        });
      })();
    </script>
  </body>
</html>
```

# flyed CMS — Editor Guide

The flyed CMS is a browser-based editor for blog content. It writes to Git and creates pull requests — no separate database.

1. Click **"New Blog (ไทย)"**

1. Click the post in the collection list
2. Make your changes

1. Edit a post

## Previewing your work

- Click the **"View"** button (top right) to see how the post will look
- The preview loads from a CF Pages branch deploy (real Astro-rendered page)
- **First preview takes 30–60 seconds** while CF Pages builds the branch
- Subsequent updates are faster

## Hero images

- Click the image field to upload
- Max 5MB per image
- Images commit to `public/images/uploads/blog/` in the repo
- Use landscape images, ideally 1600×900 or larger

# Decap CMS Operator Runbook

This document is for the developer who set up Decap CMS. Editors should read `public/admin/README.md` instead.

## One-time setup (operator only)

### 1. Decap Cloud account

1. Sign up at https://decapcms.org/cloud/ (free tier: 1 editor, unlimited repos)
2. Add a site: `https://flyed.dev`
3. Connect the GitHub repo: `flyed-dev/flyed`
4. Grant Decap Cloud access to the repo (via GitHub App)
5. Note the GitHub username(s) of authorized editors

### 2. CF Pages branch deploys (verify)

The preview iframe assumes CF Pages is configured to deploy PR branches.

1. Cloudflare Dashboard → flyed Pages project → Settings → Builds
2. Confirm "Enable branch build previews" is ON
3. Preview URL format: `https://<branch>--flyed-dev.pages.dev`

### 3. Invite an editor

1. Add the editor as a GitHub collaborator on `flyed-dev/flyed` (Write access)
2. Add their GitHub username in Decap Cloud → Team
3. Send them the editor README link: `https://flyed.dev/admin/README.md`

## How it works

```
Editor saves in /admin
  → Decap Cloud OAuth via git-gateway
  → Decap creates branch: cms/<slug>
  → Decap commits + opens PR
  → CF Pages auto-deploys branch
  → Editor previews at https://<branch>--flyed-dev.pages.dev/blog/<slug>
  → Reviewer merges PR
  → CF Pages deploys production
```

## Day-to-day operations

### Add a new editor

1. GitHub: invite as collaborator
2. Decap Cloud: add to team
3. Done. They log in at `/admin`.

### Disable editorial workflow temporarily

Edit `public/admin/config.yml`. Set `publish_mode: editorial_workflow` to a different value (or remove). Edits will commit directly to main. **Use with caution.**

### Roll back the entire CMS integration

See "Rollback" section in `docs/superpowers/specs/2026-07-03-decap-cms-integration-design.md`.

## Troubleshooting

### Editor can't log in

- Check Decap Cloud → Team → editor's GitHub username is listed
- Check editor is a GitHub collaborator on the repo
- Check `backend.repo` in `config.yml` matches the actual repo name

### Preview iframe shows 404

- Branch hasn't deployed yet — wait 30–60s
- Check CF Pages → Deployments → branch listed with success status
- Verify branch name doesn't have unsupported characters (slashes get converted to `--`)

### Editorial workflow missing (commits land on main directly)

- `publish_mode: editorial_workflow` missing from `config.yml`
- Decap defaults to direct commit if this is unset

### Image upload fails

- File > 5MB (cap enforced in widget config)
- GitHub push rejected (file too large for repo)
- Editor must resize and retry

- IP allowlist on `/admin` (Cloudflare Access) — open a PR
- Image processing pipeline — open a PR
